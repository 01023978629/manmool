# 만물인테리어 · Loop Agent

**AI가 24시간 운영하는 만물인테리어 회사 웹사이트**

상담부터 견적, 시공 관리, 사후 관리까지 — AI 루프 에이전트가 순환하며
운영하는 인테리어 회사 콘셉트의 반응형 원페이지 웹사이트입니다.

기획서(`AI 운영형 홈페이지 기획서`)의 구조 —
**웹 → n8n(워크플로) → 카카오톡(알림·챗봇) → 대표 승인** — 을 그대로 구현했습니다.

벤치마킹 비율 — **3.3디자인스페이스 운영구조 40% · 1204디자인 견적 30% · 아파트멘터리 화면 25% · 인테리어티쳐 프리미엄 5%**.
겉은 깔끔·단순하게, 내부는 **견적·고객·현장·사진·A/S 데이터가 연결되는 업무 시스템**으로 구성했습니다.

## 홈페이지 (고객 화면)

- **시공 사례 (아파트멘터리형)** — 지역·아파트명·평수·공사비·공간종류·스타일·공사범위·공사연도 구조화 필터, 아파트명 검색, **Before/After 상세**, 자재·핵심문제·해결방법, **유사 사례 추천** (AI 사진 자동분류 안내)
- **대화식 AI 예상견적** — 공간→평수→전체·부분→예산→사진을 대화로 묻고, **예상금액·필요공사·추천 자재등급·예상 공사기간·비슷한 사례·상담 전 준비**를 생성
- **면허·보증·A/S 신뢰 섹션** — 고액 계약 신뢰 확보
- **체계적·효율적 운영 프로세스 = n8n·카카오톡** — 자동화 파이프라인 + 효율 지표 시각화
- **단계형 AI 상담 문의** → n8n 웹훅 · **카카오톡 챗봇** (평수·예산·스타일 슬롯 채우기)

## 업무 시스템 (내부 화면)

| 페이지 | 내용 |
|--------|------|
| `admin.html` 관리자 | 리드·AI 요약·우선순위·**승인/보류/거절** (사람 승인 장치) |
| `mypage.html` 고객 마이페이지 | 계약·견적·일정·오늘의 공정·현장사진·**변경/추가비용 승인**·자재선택·결제·문서·준공 체크리스트 |
| `field.html` 현장관리 | 오늘 할 작업·작업 전후 사진·자재 입고·문제 보고(고위험은 사람 확인)·추가공사 요청·출입 기록·완료 체크 |
| `as.html` A/S 센터 | 하자 위치·사진 접수 → **유형 자동분류·긴급도 판단·담당 배정·방문 일정**·고객 확인 |

> 모든 화면은 **프로젝트 단위**로 데이터가 연결됩니다 (`data/project.json`). 금액·계약·발송은 대표/고객 승인 후에만 진행합니다.

## 구조

```
index.html                              홈 (고객 화면)
admin.html / mypage.html                관리자 / 고객 마이페이지
field.html / as.html                    현장관리 / A/S 센터
css/styles.css                          홈 스타일
css/admin.css / css/app.css             관리자 / 업무 시스템 공통 스타일
js/main.js                              홈 렌더링(사례·신뢰·자동화)
js/estimate.js                          대화식 AI 예상견적
js/inquiry.js                           단계형 상담 폼 → n8n 전송
js/admin.js / js/mypage.js              관리자 / 마이페이지
js/field.js / js/as.js                  현장관리 / A/S 센터
data/site.json                          홈 콘텐츠(회사·서비스·사례·견적)
data/project.json                       프로젝트 데이터(일정·사진·자재·결제·A/S)
data/config.json                        연동 설정(n8n 웹훅·카카오 채널)
integrations/INTEGRATION.md             n8n·카카오 연동 가이드
integrations/n8n/*.workflow.json        n8n 워크플로(import용)
```

## n8n · 카카오톡 연동

1. `integrations/n8n/`의 워크플로 2개를 n8n에 **Import** 합니다.
   - `manmul-inquiry.workflow.json` — 문의 접수 → AI요약 → 카카오 알림 → 대표 승인 → 고객 발송
   - `kakao-chatbot-skill.workflow.json` — 카카오 i 오픈빌더 챗봇 스킬서버
2. `data/config.json`에 n8n 웹훅 URL과 카카오 채널 URL을 넣고 `enabled: true`, `demoMode: false`로 전환합니다.
3. 자세한 절차는 [`integrations/INTEGRATION.md`](integrations/INTEGRATION.md) 참고.

> **안전장치:** 금액·계약·고객 발송은 **대표 승인 후에만** 실행됩니다(초안·발송 분리).
> AI는 요약·분류·초안까지만 담당합니다.

## 콘텐츠 운영 (AI/운영자용)

- 사이트 콘텐츠는 **`data/site.json`**, 연동 설정은 **`data/config.json`** 하나로 관리됩니다.
- 두 파일만 수정하면 코드 변경 없이 사이트·연동이 갱신되므로, AI 에이전트가 운영할 수 있습니다.

## 로컬 실행

`fetch`로 JSON을 불러오므로 로컬 서버가 필요합니다.

```bash

# http://localhost:8000 접속
```
