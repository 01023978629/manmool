# n8n 워크플로 — 어떤 걸 쓸까?

만물인테리어 상담 파이프라인은 **두 가지 버전**을 제공합니다.
**1인 운영이면 `lite`(무료) 버전으로 시작하세요.**

| 파일 | 분석 방식 | 저장 | 비용 | 추천 대상 |
|---|---|---|---|---|
| **`manmul-inquiry-lite.workflow.json`** ⭐ | **규칙 엔진 Code 노드**<br>(외부 API 없음) | **Google Sheets** | **완전 무료** | 1인 운영 · 지금 바로 시작 |
| `manmul-inquiry.workflow.json` | Claude API(유료·키 필요) | Postgres(DB 서버) | 소액 유료 | 문의량 많고 자유서술 요약이 필요할 때 |
| `kakao-chatbot-skill.workflow.json` | — | — | 무료 | 카카오 챗봇 자동응대(선택) |

## 왜 lite가 기본인가

- **분석 로직이 이미 검증됨** — 사이트 관리자 화면(`admin.html`)의 `aiSummary`
  알고리즘을 그대로 이식했습니다. 같은 상담이면 admin 화면과 n8n이 **동일한
  우선순위·점수·요약**을 냅니다. (점수 가중치 +20/+15/+10/+5, 우선순위 ≥80 높음 / ≥65 보통)
- **API 키·DB 서버가 필요 없음** — Claude 키, Postgres 접속정보 없이 import 즉시 동작.
- **비용 0원** — Google Sheets 무료, Code 노드 무료. 실제 고객 알림 발송(알림톡)만
  나중에 사업자 등록 후 소액.

## lite 워크플로 구조 (9노드)

```
Webhook(상담 수신)
  → 정규화 + 규칙 분석(무료 Code)   ← leadId·요약·추가질문·점수·우선순위·예산적합성
  → 리드 저장(Google Sheets)         ← 무료
  → 대표에게 알림(카카오/텔레그램)
  → 웹사이트에 응답
  → 대표 승인 대기(Wait)             ← 무승인 발송 방지 게이트
  → 승인 여부(If) ─┬─ 고객에게 안내 발송(승인분)
                   └─ 보류/거절(발송 안함)
```

> 나중에 문의량이 늘어 자유서술 요약이 필요하면 `manmul-inquiry`(Claude 버전)로
> 갈아타면 됩니다. 웹훅 계약(입력 필드)은 동일하니 사이트 수정 없이 교체됩니다.

설치 순서는 상위 폴더의 [`../SETUP-n8n-kakao.md`](../SETUP-n8n-kakao.md)를 따르세요.
