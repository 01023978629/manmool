## 1. 확정 결함 (3인 검증 통과 또는 감사자 직접 재확인) — 수정 권장

### 1-1. blog.html 모바일 햄버거 메뉴가 동작하지 않음 — `blog.html:34`
blog.html에는 `#navToggle` 버튼이 있지만 로드 스크립트는 `js/blog.js`뿐이고, 토글 배선
(`setupUI`)은 `js/main.js`에만 있다. 390px 실측: 클릭해도 `aria-expanded="false"` 고정,
메뉴 안 열림. 모바일에서 블로그를 본 고객이 헤더로 상담·견적에 못 간다.
**수정 방향**: blog.js에 소형 토글 배선 추가(또는 공용 nav 스크립트 분리).
검증: `grep -c navToggle js/blog.js` → 0.

### 1-2. 상담 요약에 자유입력 지역이 escape 없이 innerHTML 삽입 (self-XSS) — `js/inquiry.js:179`
`renderSummary()`의 `${v}`가 미escape. `region`은 자유 텍스트라
`<img src=x onerror=...>` 입력 시 4단계 요약에서 실행. admin 쪽은 escapeHtml로 방어돼
저장형으로 확산되진 않지만 escape 누락 자체는 확정 결함.
**수정 방향**: rows 매핑에서 값에 escapeHtml 적용 (inquiry.js에 이미 있는 esc 유틸 재사용).

### 1-3. site.json 로드 실패 시 상담 폼·AI 견적이 안내 없이 죽은 화면 — `js/main.js:764`
init()이 로드 실패 시 폴백(연락처)만 렌더하고 return → 견적 섹션은 빈 창, 상담 폼은
버튼이 보이는데 클릭해도 무반응(실측). 입력을 다 채운 고객이 이유도 모른 채 막힌다.
**수정 방향**: 실패 분기에서 두 섹션에 "일시 오류 — 전화 010-2397-8629" 안내 렌더.

### 1-4. admin이 kakao.ready:false를 무시하고 카카오 발송 UI 활성 — `js/admin.js:147`
공개 화면은 `kakao.ready && url`로 게이트하는데 admin은 URL만 확인 → 미개설 채널
버튼이 활성이고, 누르면 허위 "카카오톡 발송됨" 기록(recordSent)까지 남는다. 연동 상태
패널(334행)도 미개설 채널을 ✓ 표시. 대표가 "고객에게 연락했다"고 오인할 수 있음.
**수정 방향**: `hasKakao`에 `kakao.ready` 조건 추가 + 연동 패널에 ready 행 표시.

### 1-5. 글 상세 canonical이 서버 HTML에선 목록을 가리킴 (네이버 색인 위험) — `blog.html:9`
`?post=` URL 6개의 서버 전달 HTML은 canonical/og:url이 목록(blog.html) 고정, JS 렌더
후에만 글 URL로 교체. Googlebot은 OK지만 JS 렌더링이 불안정한 네이버 Yeti는 6개 글을
전부 중복 처리할 수 있다. 대전 영업엔 네이버가 핵심이라 실손실.
**수정 방향(택1)**: 글별 정적 페이지 프리렌더(배포 스크립트에서 생성) 권장.

### 1-6. css/js 캐시버스터 페이지별 불일치 — `admin.html:10` 외
index·blog만 `styles.css?v=...`, admin/as/field/mypage는 무버전(+hj-link.js 전부 무버전).
같은 파일이 두 캐시 엔트리로 갈라짐. 영향은 낮음(max-age 600).
**수정 방향**: 전 페이지 동일 버전 쿼리로 통일.

## 3. 운영자(대표) 액션 — 코드 아님

- **robots.txt는 프로젝트 경로에선 크롤러가 읽지 않음** (호스트 루트만 조회). noindex가
  있어 실해는 없지만 **sitemap 자동 발견이 안 되므로** 네이버 서치어드바이저·구글
  서치콘솔에 sitemap.xml **수동 제출 필수** (GROWTH.md 절차 그대로).
- index.html의 네이버·구글 소유확인 메타는 아직 주석 상태 — 코드 발급받아 전달 필요.
