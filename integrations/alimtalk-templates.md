# 카카오 알림톡 템플릿 3종 (심사 제출용 초안)

만물인테리어 자동 발송(승인 후 n8n → 알림톡)에 쓰는 **정보성** 알림톡 템플릿입니다.
아래 문구를 카카오 비즈메시지(발신프로필) 관리자 또는 대행 파트너(솔라피/알리고/NHN 등)의
**템플릿 등록**에 붙여넣고 심사를 올리세요. 승인되면 각 템플릿에 **템플릿 코드**가 부여됩니다.

> **원칙:** 이 셋은 고객이 먼저 문의를 시작했을 때 나가는 **정보성** 메시지입니다(광고 아님).
> **T1(접수 확인)** 만 시스템 자동, **T2·T3** 은 반드시 **대표 승인 후** 발송합니다
> (무승인 발송 0 · 돈·일정은 승인·기록을 거친다).

---

## 공통 규칙 (심사 통과 팁)

- 변수는 `#{변수명}` 형식. 변수만으로 시작/종료하거나, 변수만으로 구성된 문장은 반려됩니다.
- 정보성 메시지에는 광고 문구(할인/이벤트/최저가 등) 금지. 넣으려면 별도 **친구톡**으로.
- 채널 홍보/유입 문구는 본문이 아니라 **버튼**으로 처리합니다.
- 야간(21~08시) 정보성 발송은 예외 인정되나, 실무상 발송 시각은 n8n에서 제어하세요.
- 발신프로필(채널)과 사업자 정보가 일치해야 합니다. (만물인테리어 / 895-48-01132)

## T1. 상담 접수 확인  — `manmul_intake`
**분류:** 정보성 · 시스템 자동(폼 제출 직후)

```
[만물인테리어] 상담 접수 안내

#{고객명}님, 문의해 주셔서 감사합니다.
아래 내용으로 상담이 접수되었습니다.

▶ 공간: #{공간요약}
▶ 접수번호: #{접수번호}

담당자가 확인 후 예상 범위와 실측 일정을
카카오톡으로 안내드리겠습니다.
```

- 변수: `고객명`, `공간요약`(예: 대전 유성구 34평 전체), `접수번호`(INQ-...)
- 버튼: ① `채널 추가`(채널 홈) ② `상담 현황 보기`(웹링크, 웹링크 URL = 고객 마이페이지)

---

## T2. 실측 일정 제안  — `manmul_survey`
**분류:** 정보성 · **대표 승인 후 발송**

```
[만물인테리어] 방문 실측 일정 안내

#{고객명}님, 요청하신 #{공간요약} 건
방문 실측 일정을 제안드립니다.

▶ 제안 일시: #{제안일시}
▶ 소요 시간: 약 #{소요시간}

일정 조정이 필요하시면 아래 버튼으로 알려주세요.
정확한 금액·자재는 실측 후 확정됩니다.
```

- 변수: `고객명`, `공간요약`, `제안일시`, `소요시간`(예: 30분)
- 버튼: ① `일정 확정/변경`(웹링크, 고객 마이페이지의 일정 패널) ② `전화 문의`(전화걸기 010-2397-8629)

---

## T3. 예상 범위·진행 안내  — `manmul_estimate`
**분류:** 정보성 · **대표 승인 후 발송**

```
[만물인테리어] 예상 범위 안내

#{고객명}님, #{공간요약} 건 검토 결과입니다.

▶ 참고 예상 범위: #{예상범위}
▶ 다음 단계: #{다음단계}

위 금액은 유사 조건 기준 참고값이며,
정확한 견적은 현장 실측 후 서면으로 확정됩니다.
계약·결제는 대표 확인과 기록을 거쳐 진행됩니다.
```

- 변수: `고객명`, `공간요약`, `예상범위`(예: 3,800만~4,300만원), `다음단계`(예: 방문 실측 예약)
- 버튼: ① `자세히 보기`(웹링크, 고객 마이페이지) ② `상담 채팅`(채널 1:1)

## 발송 매핑 (n8n)

두 워크플로(`manmul-inquiry`, `manmul-inquiry-lite`)의 **"고객에게 안내 발송 (승인분)"**
노드에는 템플릿 코드가 **`manmul_survey`(T2)로 직접 적혀** 있습니다. 다른 템플릿으로
보내려면 그 노드의 `templateCode` 값을 아래 표의 코드로 바꾸세요. (심사에서 승인받은
코드와 글자까지 같아야 발송됩니다 — 미등록 코드는 발송 대행사에서 반려됩니다.)

| 상황 | 템플릿 | 승인 필요 | 현재 워크플로 |
|---|---|---|---|
| 폼 제출 직후 접수 확인 | `manmul_intake` (T1) | 자동 발송 가능 | 미구현(선택) — 원하면 Webhook 직후에 발송 노드 추가 |
| 실측 일정 잡을 때 | `manmul_survey` (T2) | ✅ 대표 승인 | ✅ 승인분 발송 노드에 연결됨 |
| 예상 범위 안내 | `manmul_estimate` (T3) | ✅ 대표 승인 | `templateCode`를 이 코드로 교체 시 사용 |

> 실제 API 키·발신프로필 키(senderKey)는 **repo에 넣지 말고** n8n 자격증명/환경변수로 관리하세요.
> `config.json` 에는 **템플릿 코드와 발송 제공자 이름**처럼 비밀이 아닌 값만 둡니다.
