# 자율 운영 루프 — 가동 가이드

> 판단은 **AI**, 몸은 **사람**, 연료는 **토큰**.
> 이 문서는 [AI 운영 헌장](./AI-OPERATING-MODEL.md) §2·§3을 **실제 구현**과 연결한다.

## 1. 지금 도는 것 (SENSE → DECIDE)

전자계약 백엔드(`contract-backend`)가 서버 상태를 읽어 **'다음 한 수'를 규칙 기반으로 판단**하고 자율 등급을 부여한다. **아무것도 발송·실행하지 않는다** — 판단만.

| 단계 | 구현 | 설명 |
|---|---|---|
| **SENSE** 감지 | `listContracts`·`listReceivables`·`financeSummary` | 계약 상태·미수·재무를 읽는다(VOID 계약 제외) |
| **DECIDE** 결정 | `operatorBrief()` → `GET /api/operator/brief` | 현장·미수별 '다음 한 수'를 판단·우선순위화, 등급 부여 |
| **주간 요약** | `weeklyReport()` → `GET /api/operator/weekly` | 지난 7일 신규·완료·입금·미수·승인대기 + 읽을 수 있는 텍스트 리포트 |
| **화면** | `/admin` 🧠 오늘의 운영 판단 · 📈 주간 요약 | 승인 대기 강조·등급 배지·리포트 복사 |

## 2. 자율 등급 (`src/autonomy.mjs`)

모든 결정에 등급이 붙는다. **미등록 행동은 가장 보수적 HUMAN 기본**(정체불명 행동의 승인후 실행 차단).

| 등급 | 규칙 | 예 |
|---|---|---|
| **AUTO** | AI 단독 | 아침 브리핑·리드 팔로업 초안·상태 정리 |
| **NOTIFY** | AI 실행 + 통보 | 진행상황 알림·일정 확정 안내(정보성) |
| **APPROVE** | 대표 승인 후 | 서명 리마인드·대금 청구·독촉·견적 발송(고객 행동요구·돈) |
| **HUMAN** | 사람만 | 물리 시공·법적 서명·은행 이체·세무 신고 |

## 3. 현재 생성되는 결정 규칙

- 발송/열람 후 오래된 미서명(`updated_at` 기준 ≥2일) → **서명 리마인드**(APPROVE)
- 완료됐으나 대금 일정 없음 / 미청구 대금 → **청구**(APPROVE)
- 청구했으나 미입금 → **입금 확인·독촉**(APPROVE)
- 취소(VOID) 계약은 어떤 결정·미수·재무에도 들어가지 않는다(취소 고객 독촉 방지)

## 4. 루프를 스케줄로 돌리려면 (상시 운영자 — 다음 단계)

지금은 **사람이 `/admin`을 열 때** 판단이 갱신된다. "토큰만 주면 도는" 상태로 가려면 브리핑을 **주기적으로 자동 기동**해야 한다.

가동 옵션(대표 옵트인 후):
1. **AI 세션 스케줄(Routine)** — 매일 아침 AI가 깨어나 `GET /api/operator/brief`·`/weekly`를 읽고, AUTO는 실행, NOTIFY/APPROVE는 대표에게 요약해 알린다.
2. **크론 + 웹훅** — 서버 크론이 브리핑을 계산해 승인 대기 항목을 대표 채널로 push.

> **정직한 경계(반드시 지킴):**
> - 브리핑·리포트는 **판단·집계만**. 실제 발송은 **APPROVE 승인** 또는 기존 발송 라우트(`/api/notify`, 대금 청구)로만.
> - **고객 발송·독촉·청구는 대표 승인 후**에만. 무승인 고객 발송 0.
> - **돈·법·물리는 사람.** 전화번호 등 PII는 브리핑·리포트에 싣지 않는다(마스킹조차 응답 제외).
> - 라이브 스케줄(실제 고객 접촉 자동화)은 **카카오 템플릿 승인 + 대표 옵트인** 이후에만 켠다.
