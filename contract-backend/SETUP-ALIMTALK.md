# 실제 알림톡 발송 셋업 가이드 (사장님용)

알림톡은 카카오가 API를 직접 주지 않고 **공식 대행사**(여기선 솔라피)를 통해 발송합니다.
아래 1~5는 **사장님이 직접** 하셔야 하는 계정·심사 작업이고, 6~7은 준비되면 서버에 연결하는 단계입니다.

> ⚠️ 순서 중요: **템플릿이 카카오 심사를 통과해야** 실제 발송이 됩니다(보통 영업일 1~3일).
> 승인 전·본인번호 테스트 전에는 실제 고객에게 발송하지 않습니다.

---

## 1. 카카오 비즈니스 채널 만들기
1. 카카오톡 채널 관리자센터(business.kakao.com)에서 **채널 개설**
2. **비즈니스 인증**(사업자등록번호 895-48-01132) 완료 — 알림톡은 인증 채널만 가능

## 2. 솔라피(Solapi) 가입 + 발신번호 등록
1. solapi.com 가입(사업자)
2. **발신번호(문자 대체발송용)** 등록·인증 — 회사 대표번호(예: 사무실/휴대폰)
3. 충전(알림톡 건당 십원대, 문자 대체발송은 별도 단가)

## 3. 카카오 발신프로필 연결
1. 솔라피 콘솔 → **카카오 알림톡 → 발신프로필 등록**
2. 1번에서 만든 카카오 채널을 연결(채널 검색용 ID 또는 관리자 인증)
3. 발급되는 **pfId(발신프로필 키)** 를 메모

## 4. 알림톡 템플릿 등록·심사
1. 솔라피 콘솔 → **알림톡 템플릿 등록**
2. [`templates/alimtalk-templates.md`](./templates/alimtalk-templates.md) 의 두 문안(`contract_sign`, `contract_done`)을 각각 등록
   - 본문·버튼(웹링크)·변수(`#{...}`)를 문서 그대로 입력
3. **심사 요청** → 승인되면 각 템플릿의 **템플릿 ID** 발급(메모)

## 5. API 키 발급
1. 솔라피 콘솔 → **개발/연동 → API Key** 발급
2. **API Key** 와 **API Secret** 을 안전하게 보관(외부 유출 금지)

---

## 6. 서버에 연결 — ① 웹 관리자 페이지(권장)

셸을 몰라도 **브라우저에서** 연결 정보를 넣을 수 있습니다.

1. 서버에 **관리자 토큰**만 환경변수로 한 번 설정: `export ADMIN_TOKEN=$(openssl rand -hex 16)`
   (이게 없으면 관리자 페이지가 비활성화됩니다)
2. `http://<서버주소>/admin` 접속 → 관리자 토큰으로 로그인
3. **Solapi 연결 정보**(API Key/Secret·pfId·발신번호·템플릿 ID) 입력 후 저장
   - 저장값은 서버 로컬 DB에 보관되고 **즉시 발송에 반영**됩니다(재시작 불필요)
   - 시크릿은 저장 후 화면에 원문을 다시 보여주지 않습니다(끝 4자리만)
4. **실제 발송 켜기**를 `1`로 저장 → 상태가 "ON · 준비완료"가 되면 연결 완료
5. **본인번호 테스트 발송**으로 확인(아래 7번)

> `CONTRACT_PEPPER` 는 서버 시크릿이라 관리자 페이지가 아니라 **환경변수로만** 설정합니다:
> `export CONTRACT_PEPPER=$(openssl rand -hex 32)`

## 6-B. 서버에 연결 — ② 환경변수(대안)

웹 대신 환경변수로 넣어도 됩니다(값이 하나라도 빠지면 서버가 기동을 거부해 오발송을 막습니다).

```bash
export ALIMTALK_LIVE=1                     # 실제 발송 켜기(이게 없으면 항상 Mock)
export SOLAPI_API_KEY=...                  # 5번
export SOLAPI_API_SECRET=...               # 5번
export SOLAPI_PF_ID=...                    # 3번 pfId
export SOLAPI_SENDER=029302266             # 2번 발신번호(숫자만)
export SOLAPI_TEMPLATE_SIGN=...            # 4번 contract_sign 템플릿 ID
export SOLAPI_TEMPLATE_DONE=...            # 4번 contract_done 템플릿 ID
export CONTRACT_PEPPER=$(openssl rand -hex 32)   # 해시 페퍼(운영 필수)
# 선택: export SOLAPI_DISABLE_SMS=1        # 알림톡 실패 시 문자 대체발송 끄기(기본은 켬)
```

값이 하나라도 빠지면 서버가 **기동을 거부**합니다(오발송 방지).

## 7. 본인번호로 발송 테스트 (고객 아님)

실제 고객에게 보내기 전에 **사장님 본인 번호로만** 한 건 테스트합니다.

- **웹 관리자**: `/admin` 페이지의 "본인번호 테스트 발송"에 본인 번호 입력 → 발송
- **또는 셸**:
```bash
# 위 6-B 환경변수를 설정한 셸에서:
node bin/selftest-send.mjs 01012345678     # ← 사장님 본인 휴대폰 번호
```

- 이 스크립트는 **본인확인용 테스트 계약**을 만들고 `contract_sign` 알림톡 1건을 그 번호로 보냅니다.
- 카톡으로 도착 + 버튼→서명화면이 열리면 정상입니다.
- 문제 없으면, 이후 실제 계약 발송은 현장 앱/대시보드의 "알림톡 보내기"에서 진행합니다.

## 자주 막히는 곳
- **템플릿 반려**: 변수만으로 된 문장·광고 표현·채널 정보 불일치가 원인. 문서 문안 그대로 등록하면 대부분 통과.
- **발송 실패(FAILED)**: 잔액 부족 / 미승인 템플릿 ID / 수신자 채널 차단. 서버 로그의 사유 코드 확인.
- **번호 불일치 차단(PHONE_MISMATCH)**: 발송 시 넘긴 번호가 계약 당사자와 다르면 서버가 막습니다(오발송 방지).
- **비용**: 알림톡 실패 시 문자 대체발송이 되면 문자 단가가 청구됩니다. 원치 않으면 `SOLAPI_DISABLE_SMS=1`.
